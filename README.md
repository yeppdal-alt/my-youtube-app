# 유튜브 댓글 분석 앱

유튜브 영상 링크만 넣으면 댓글을 가져와서, 자주 나온 단어를 그래프와 워드클라우드로 보여주는 스트림릿(Streamlit) 앱입니다.

## 파일 구조

```
.
├── main.py                     # 메인 페이지: 링크 입력 → 댓글 수집/정렬 → 단어 빈도 그래프
├── pages/
│   └── 2_워드클라우드.py         # 워드클라우드 페이지 (사이드바에 자동으로 표시됨)
└── requirements.txt             # 필요한 라이브러리 목록
```

스트림릿은 `pages` 폴더 안의 파일을 자동으로 인식해서 사이드바 메뉴에 추가하기 때문에, 별도 설정 없이 두 페이지가 앱에 함께 나타납니다.

## 단계별 기능

### 1단계 — 댓글 불러오기 (`main.py`)
- 유튜브 영상 링크 입력창 (기본값: 딥마인드 다큐 영상)
- 예시 버튼 2개로 미리 준비된 링크 채우기 (영어 댓글 / 한국어 댓글)
- `youtu.be`, `youtube.com/watch` 두 형태의 링크에서 영상 ID 자동 추출 (`si=` 등 부가 파라미터는 무시)
- YouTube Data API v3의 `commentThreads`로 댓글 최대 100개 요청 (`order=relevance`로 좋아요 많은 순)
- 가져온 댓글을 좋아요 수 기준 내림차순 정렬
- 댓글 개수는 지표 카드로, 댓글 목록은 표로 표시
- 링크가 잘못됐거나 댓글이 막힌 영상이면 한국어 안내 메시지 표시

### 2단계 — 자주 나온 단어 TOP 20 (`main.py`)
- 댓글 전체를 단어 단위로 분리 (한 글자짜리 단어는 제외)
- 등장 횟수 상위 20개 단어를 집계
- Plotly 가로 막대그래프로 시각화 (가장 많이 나온 단어가 맨 위)

### 3단계 — 워드클라우드 (`pages/2_워드클라우드.py`)
- 메인 페이지에서 불러온 댓글(세션에 저장됨)을 재사용
- 한 글자짜리 단어를 제외하고 워드클라우드 생성 (배경 흰색)
- 한글이 깨지지 않도록 나눔고딕(NanumGothic) 폰트를 GitHub에서 다운로드해 적용
- matplotlib 없이 `wordcloud` 라이브러리가 생성한 이미지를 바로 화면에 표시
- 폰트 다운로드에 실패하면 한국어 안내 메시지 표시
- 메인 페이지에서 댓글을 아직 불러오지 않았다면 안내 메시지 표시

## 실행 방법 (로컬)

```bash
pip install -r requirements.txt
streamlit run main.py
```

## 스트림릿 클라우드 배포 시 설정

1. 이 저장소(또는 폴더)를 GitHub에 올리고 스트림릿 클라우드에서 앱 생성
2. **Settings → Secrets**에 아래 내용 추가

   ```toml
   YOUTUBE_API_KEY = "발급받은_YouTube_Data_API_키"
   ```

3. YouTube Data API v3 키는 [Google Cloud Console](https://console.cloud.google.com/)에서 프로젝트를 만들고 "YouTube Data API v3"를 사용 설정한 뒤 발급받을 수 있습니다.

## requirements.txt

```
requests
plotly
wordcloud
```

(streamlit, pandas, numpy는 스트림릿 클라우드에 기본 포함되어 있어 목록에서 제외했습니다.)

## 알아두면 좋은 점

- 워드클라우드 페이지는 메인 페이지에서 댓글을 먼저 불러와야 데이터를 볼 수 있습니다 (세션 상태로 댓글을 공유).
- 폰트 파일은 세션당 한 번만 다운로드하도록 캐시되어 있어, 페이지를 오갈 때마다 다시 받지 않습니다.
- YouTube API는 일일 할당량(쿼터)이 있어, 너무 많이 요청하면 일시적으로 댓글을 못 가져올 수 있습니다.
